01 — The problem

RNNs read one word
at a time

Before this paper, translation models (RNNs / LSTMs) processed a sentence sequentially — word 1, then word 2, then word 3 — because each step depends on the hidden state from the step before.

Why that's bad: you can't parallelize it (step 5 needs step 4 to finish first), and by the time you reach word 40, the model has half-forgotten word 2. Long-range dependencies get weaker with distance.
sequential processing — one path, in order
↩ agenda
self-attention — every word looks at every word, at once
02 — The idea

Delete the recurrence.
Keep only attention.

The Transformer throws out recurrence and convolutions. Instead, every position in a sentence directly attends to every other position, in a single step — no waiting in line.

fully parallelizable constant path length O(1) sequential ops
↩ agenda
03 — The mechanism

Scaled Dot-Product Attention

Think of it like a library search. Every word produces three vectors:

  • Query (Q) — "what am I looking for?"
  • Key (K) — "what do I contain, so others can find me?"
  • Value (V) — "what information do I actually offer?"

Compare a word's Query against every other word's Key to get a relevance score, turn those scores into weights with softmax, then take a weighted average of everyone's Value.

Divide by √dk before the softmax — otherwise the dot products get so large the softmax gradients vanish. That's the "scaled" part.
the whole mechanism, as one formula
Attention(Q,K,V) = softmax( QKT / √dk ) V

04 — Seeing it work

Attention weights are
literally interpretable

In a sentence, the word it is ambiguous — what does it refer to? Self-attention lets the model learn to assign high weight from it back to whichever noun it actually refers to, purely from data.

This grid is the actual object being computed inside every attention layer: a full matrix of "how much should token i listen to token j."

05 — Scaling it up

Multi-Head Attention: 8 lenses, not 1

One attention "head" tends to average things out and settle on a single pattern. So instead of computing attention once, the Transformer projects Q, K, V into 8 smaller subspaces and runs attention 8 times, in parallel.

Each head is free to specialize — one might track subject–verb agreement, another coreference (like it → the animal), another something purely syntactic. The 8 results get concatenated and projected back down.

h = 8 headsd_k = d_v = 64d_model = 512
Used in 3 places: encoder self-attention, masked decoder self-attention, and encoder–decoder attention (decoder queries, encoder keys/values).

06 — Putting it together

The full encoder–decoder stack

Encoder (left): 6 identical layers. Each has self-attention, then a feed-forward network — with a residual connection + layer norm around each.

Decoder (right): also 6 layers, but with an extra sub-layer that attends over the encoder's output — plus its self-attention is masked so position i can only see positions before it.

Every sub-layer's output is LayerNorm(x + Sublayer(x)) — the residual connection is what makes 6-deep stacks trainable at all.

09 — The other sub-layer

Feed-Forward Networks & shared embeddings

Every encoder/decoder layer also has a position-wise feed-forward network — the same two-layer MLP applied independently to each token: a linear layer, ReLU, then another linear layer.

FFN(x) = max(0, xW1+b1)W2+b2
d_model = 512d_ff = 2048same weights per position

Input/output tokens are converted to vectors with learned embeddings. The paper shares one weight matrix across the input embedding, output embedding, and the pre-softmax linear layer — then scales embeddings by √d_model.

Think of the FFN as where each token "thinks alone" after attention let it "talk" to the rest of the sentence.

07 — One catch

But attention has no
sense of order

Ditching recurrence means the model has no built-in notion of word order — "dog bites man" and "man bites dog" would look identical to pure attention. So the paper injects position information directly into the input embeddings.

Using fixed sine/cosine waves of different frequencies for each dimension — not learned, so the model can (in theory) generalize to sentence lengths it never saw in training.
PE(pos, 2i) = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d)

08 — The payoff

Why self-attention wins on paper

Layer typeComplexity / layerSequential opsMax path length
Self-AttentionO(n² · d)O(1)O(1)
RecurrentO(n · d²)O(n)O(n)
ConvolutionalO(k · n · d²)O(1)O(log_k n)
Self-Attention (restricted)O(r · n · d)O(1)O(n/r)

Constant sequential operations means full parallelization across a training batch. Constant path length means a signal from word 1 reaches word 100 in a single hop — recurrence needs 99.

10 — Making it train

Data, optimizer, regularization

Data: WMT 2014 English-German (4.5M sentence pairs, byte-pair encoded, ~37K shared vocab) and the larger English-French set (36M sentences, 32K word-piece vocab).

Hardware: 8 NVIDIA P100 GPUs. Base model: 100K steps / ~12 hours. Big model: 300K steps / 3.5 days.

Optimizer: Adam (β₁=0.9, β₂=0.98) with a learning rate that warms up linearly for 4000 steps, then decays ∝ step-0.5.

Regularization: residual dropout (P=0.1) after every sub-layer, plus label smoothing (0.1) — deliberately making the model less confident to improve BLEU, even though it hurts perplexity.

13 — TL;DR for Q&A

If you remember five things

1. Recurrence is replaced entirely by self-attention — every token attends to every other token in one step.

2. Attention = weighted average of Values, where weights come from comparing Queries against Keys, scaled by √d_k.

3. Multiple attention heads (8 here) let the model track different kinds of relationships in parallel.

4. Since attention has no notion of order, sinusoidal positional encodings are added to the input embeddings.

5. The payoff: full parallelization, shorter dependency paths, better translation quality, far less training compute.

This architecture — encoder + decoder + multi-head attention — is the direct ancestor of BERT, GPT, and effectively every large language model that followed.
